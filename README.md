# ScholarAIO — Research OS Core

ScholarAIO is a local literature engine for researchers: ingest PDFs, search by keyword or semantics, cluster topics, query citation graphs, and export BibTeX — all from one CLI. This repo is the share edition, providing the full engine, Obsidian vault templates, and operational scripts without any private data.

> **Based on** [ScholarAIO](https://github.com/zimoliao/scholaraio) by [Zi-Mo Liao](https://zimoliao.github.io/), licensed under MIT. This repo extends the original with Claude Code skills, multi-model pipelines, and Obsidian vault integration.

## Why This Exists

Researchers juggle scattered PDFs, disconnected notes, and one-off scripts. ScholarAIO turns a local paper collection into a searchable, analyzable knowledge base you can query in natural language — without uploading anything to the cloud.

## Core Features

- **Hybrid search** — keyword (FTS5) + semantic (FAISS) + fusion ranking
- **Layered reading** — L1 metadata, L2 abstract, L3 conclusions, L4 full text
- **PDF / Markdown ingest** — MinerU conversion, metadata extraction, deduplication
- **Topic modeling** — BERTopic clustering with interactive visualization
- **Citation graph** — references, citing papers, shared-reference analysis via OpenAlex
- **Workspace management** — paper subsets per research project
- **BibTeX export** — all papers, by workspace, or by filter
- **Natural language Q&A** — `scholaraio ask "..."` routes to search, SQL, or KG automatically

## Who Is This For

- Researchers managing 100+ papers locally
- Students writing literature reviews
- Anyone who wants a private, searchable knowledge base without SaaS lock-in

## Quick Start

**Requirements:** Python 3.10+, Windows 10/11 (PowerShell 5.1+), Git

### 1. Clone and bootstrap

```powershell
git clone <repo-url>
cd research-os-core
powershell -ExecutionPolicy Bypass -File .\bootstrap.ps1 -WithDemoData
```

This creates local directories (`data/`, `workspace/`, `vault/`), copies demo data, and generates config files.

### 2. Install

```powershell
pip install -e .
```

### 3. Configure API keys

```powershell
# Edit the generated config.local.yaml:
notepad config.local.yaml
```

Fill in at minimum:
- `llm.api_key` — for metadata extraction, Q&A, and enrichment (OpenAI-compatible)

See [Configuration](#configuration) below for which features need which keys.

### 4. First commands

```bash
# See all available commands
scholaraio --help

# Run a benchmark (works with demo data, no API key needed)
scholaraio benchmark

# Search your library
scholaraio usearch "phase separation thermodynamics"

# Ask a question in natural language
scholaraio ask "what methods are used to study liquid-liquid phase separation?"

# View a paper at different depths
scholaraio show "AuthorName-2024" --level L2

# Cluster your library by topic
scholaraio topics

# Export references
scholaraio export bibtex --all -o refs.bib
```

## Configuration

Two config files, both gitignored:

| File | Source | Purpose |
|------|--------|---------|
| `config.yaml` | Generated from `config.yaml.example` by bootstrap | Paths, model names, feature toggles |
| `config.local.yaml` | Generated by bootstrap as placeholder | API keys and secrets |

### What needs API keys, what doesn't

| Feature | Requires | Without key |
|---------|----------|-------------|
| Keyword search (`search`) | Nothing | Full function |
| Semantic search (`vsearch`, `usearch`) | Embedding model (auto-downloads on first use) | Full function |
| Benchmark | Nothing | Full function |
| Topic modeling (`topics`) | Embedding model | Full function |
| BibTeX export (`export`) | Nothing | Full function |
| Natural language Q&A (`ask`, `nlsearch`) | LLM API key (`llm.api_key`) | Not available |
| Metadata extraction (`pipeline`) | LLM API key | Partial (falls back to PDF parsing) |
| PDF conversion | MinerU API key (`mineru_api_key`) or local MinerU | Falls back to basic extraction |
| Citation data (`refs`, `citing`) | Nothing (uses OpenAlex, free) | Full function |
| Knowledge graph (`kg`) | LLM API key for build | Query works without key if KG already built |
| Enrichment (`enrich-l3`, `enrich-toc`) | LLM API key | Not available |

### Embedding model

On first run of any vector operation, the embedding model (~400 MB) downloads automatically from ModelScope/HuggingFace. No API key needed, but requires internet for the first download.

## Data and Privacy

- **All user data stays local.** Papers, indexes, workspaces, and notes live in gitignored directories.
- `data/` — your papers and search indexes
- `workspace/` — your active research workspaces
- `config.local.yaml` — your API keys
- `vault/` — your Obsidian vault (initialized from `vault-template/`)
- Demo data in `demo-data/` is public and separate from your private library.
- Nothing is uploaded anywhere unless you explicitly configure an external service.

## Level V Operations

Optional automated health monitoring:

- **Health daemon** — 7 checks + 4 auto-repairs + lock + cooldown + verification
- **Status products** — `system_manifest.json`, `system_health.json`, `recommended_actions.json`
- **Scheduler** — Windows Task Scheduler integration (startup + 07:30 + 23:30)

```powershell
# Manual run
powershell -ExecutionPolicy Bypass -File .\scripts\run_health_daemon.ps1

# Register scheduled tasks (requires Administrator)
powershell -ExecutionPolicy Bypass -File .\scripts\register_health_daemon_tasks.ps1
```

## Current Status

Known limitations:
- Windows-primary (PowerShell scripts; Python core works cross-platform)
- Demo data contains empty paper stubs, not full-content samples
