# 深度夜间复盘

这个复盘的价值在于"跨日期看模式"——单日复盘容易被当天情绪主导，周维度才能看出哪些目标真的在推进、哪些一直被推迟。

## 执行步骤

### 1. 全库扫描

扫描 `02_Research_Projects/`、`03_Theoretical_Work/`、`04_Learning/` 本周（最近7天）修改过的文件，按项目分类列出。这是"实际工作量"的客观证据。

### 2. 对照周目标做诚实评估

读取 `01_Planning/weekly_goals.md`，对每个目标给出状态：
- ✅ 完成
- 🔄 进行中（说明到了哪一步）
- ❌ 未启动（说明原因：优先级被挤压？遇到障碍？主动放弃？）
- 🚫 需要放弃或推迟（说明理由）

识别模式：哪类任务总被推迟？是时间估算问题、外部依赖，还是动力问题？

### 3. 提炼本周最有价值的进展

从修改文件和 `01_Planning/process_notes.md` 中提取：
- 实验数据突破或重要观察
- 理论推导的关键步骤
- 文献阅读的核心发现
- FPGA/技术学习的里程碑

### 4. 重构下周计划

更新 `01_Planning/weekly_goals.md`：
- 删除或推迟确实不该做的目标
- 补入本周发现的新的高优先级事项
- 保持下周 Top 3 聚焦，不要列超过7个目标

如果有重大研究路线调整，在 `01_Planning/process_notes.md` 中记录决策和理由。

### 5. 生成周复盘报告

写入 `01_Planning/archive/review_YYYY-MM-DD.md`，包含：
- 本周成果摘要（3–5条）
- 未完成分析（模式而非清单）
- 下周 Top 3 目标
- 给自己的一句话（诚实的，不是鸡汤）
