# 图片分析存档

图片无法直接写入 Obsidian，但分析结论可以。这个 skill 的价值是：把转瞬即逝的图片内容转化为可检索、可关联的文字记录，沉淀进研究笔记体系。

## 执行步骤

### 1. 确认图片

列出本次对话中用户发送的所有图片，注明初步类别：
- 细胞/显微镜照片
- FPGA 波形 / 逻辑分析仪截图
- 理论推导手稿 / 公式
- 文献图表 / 数据图
- 其他

### 2. 推断保存位置

根据内容映射到目标文件：

| 图片类型 | 默认目标路径 |
|---------|------------|
| 细胞实验照片 | `02_Research_Projects/Phase_Separation/` 或 `Organoid_MEA_Chip/` |
| FPGA / 波形 | `04_Learning/FPGA/` |
| 手写公式 / 推导 | `03_Theoretical_Work/` |
| 文献图表 | 对应项目文献笔记 |

如果用户在参数或对话中指定了路径，优先使用用户指定的。若仍不确定，询问用户再继续。

### 3. 分析图片内容

提取：
- **核心观察或数据**：数值、现象、结论（越具体越好）
- **与当前项目的关联**：验证了什么假设？发现了什么问题？推进了哪个任务？
- **待跟进事项**：如有

### 4. 追加写入目标 `.md` 文件

使用以下模板，追加到目标文件末尾（若文件不存在则新建）：

```markdown
## 图片记录 · YYYY-MM-DD
**来源**：[图片描述，如"EGFP-G3BP1 显微镜截图"、"ILA 计数器波形"]
**分析**：[2–5 句话，核心内容，具体数值]
**关联**：[与哪个实验/模型/任务相关]
**待跟进**：[如有，否则省略]
```

### 5. 告知用户

说明写入了哪个文件的哪个位置，展示写入内容片段供确认。
